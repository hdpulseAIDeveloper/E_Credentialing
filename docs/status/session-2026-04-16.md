### Type safety + code quality

- Extracted `src/server/services/provider-status.ts` (single source of
  truth for the status state machine) and refactored `providerRouter`
  to use it with `z.nativeEnum`. Tests: `tests/unit/server/services/provider-status.test.ts` (13).
- Removed every remaining `Record<string, unknown>` where-clause and
  `as ProviderStatus` cast from the `provider` and `report` routers;
  replaced with `Prisma.*WhereInput` and `z.nativeEnum` inputs.

### Bugs fixed along the way

1. `src/lib/blob-naming.ts::documentBlobPath` — filenames with no dot
   fell through to `.untitled` because `.split('.').pop()` never
   returns empty. Now uses `lastIndexOf('.')` with a real `.pdf` fallback.
   (Discovered by `tests/unit/lib/blob-naming.test.ts`, 11 tests.)
2. `src/lib/auth/provider-token.ts` — the `catch` block re-threw
   every error as `invalid_or_expired_token`, silently erasing the
   `invalid_token_type` signal for non-magic-link tokens. Fixed to
   re-raise `ProviderTokenError` instances unchanged.

### Observability foundation

- Structured `pino` logger with PHI-redaction paths is enforced by test.
- `/api/live` is a pure process probe (never touches DB/Redis).
- `/api/ready` returns 503 when DB or (configured) Redis is unreachable.

## Production deploy checklist

Do **not** run until the server admin confirms production is idle
(B-001 resolved).

1. Ensure `/var/www/E_Credentialing/.env` has `AUDIT_HMAC_KEY` set to
   a stable, 32+ character secret. Rotating this value will make every
   pre-rotation audit row fail `verifyAuditChain`. (B-010.)
2. Ensure the `ALLOW_DEPLOY=1` env var is exported on the dev machine
   that runs `.claude/deploy.py`. The deploy script hard-stops otherwise.
3. Run `python .claude/deploy.py` as usual; `scripts/web-entrypoint.sh`
   will run `prisma migrate deploy` and apply
   `20260416130000_audit_tamper_evidence` and `20260416140000_ncqa_catalog`.
4. If this is the *first* deploy against a DB that was bootstrapped via
   `prisma db push`, apply B-009a's `prisma migrate resolve` commands
   **before** step 3.
5. Run the smoke checks from `docs/runbooks/smoke-test.md`.
6. (Compliance) once NCQA content (B-006) arrives:
   ```bash
   python .claude/deploy.py "docker cp ncqa.csv ecred-web-prod:/tmp/ncqa.csv"
   python .claude/deploy.py "docker exec ecred-web-prod npx tsx scripts/import-ncqa-criteria.ts /tmp/ncqa.csv"
   ```
