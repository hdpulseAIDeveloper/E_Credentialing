# E2E Test Plan

End-to-end tests run against a fully wired application (web + worker + Postgres + Redis + a fake Azure via Azurite). They validate real user journeys in a real browser.

## Stack

- Playwright (chromium + firefox).
- Axe for accessibility assertions on every page visited.
- Storybook test-runner for component-level interaction coverage.

## Pre-auth fixtures

- `adminStorageState.json` — signed in as an Admin.
- `managerStorageState.json` — signed in as a Credentialing Manager.
- `specialistStorageState.json` — signed in as a Credentialing Specialist.
- `committeeStorageState.json` — Committee Member.

All are generated by a `global-setup.ts` that logs in via seeded credential pairs against a dev-only credentials provider.

## Journeys

### Staff

- **Onboard a provider end-to-end**
  1. Manager creates a provider via the "New Provider" flow.
  2. System sends the invite (captured via MSW).
  3. Specialist reviews the new provider page and sees expected state.
  4. Specialist triggers license and DEA bots; both complete (via mock external pages).
  5. Specialist marks PSV complete and sends to committee.

- **Committee session**
  1. Manager generates the agenda PDF.
  2. Chair opens the Committee Dashboard for the session.
  3. Chair records decisions on each provider.
  4. Chair finalizes the session.
  5. Minutes are attested.

- **Monthly roster**
  1. Roster manager opens the roster draft for a payer.
  2. Reviews flagged providers and resolves them.
  3. Submits; system records the acknowledgement.

- **Recredentialing bulk kick-off**
  1. Manager selects all providers due in the next 90 days.
  2. Starts recred cycles in bulk.
  3. Providers appear on the pipeline.

### Provider (no session)

- **Complete application via magic link**
  1. Provider clicks the invite link; token validates.
  2. Provider fills each section; sections save.
  3. Provider uploads required documents.
  4. Provider attests; token is revoked.

- **IDOR resistance** (negative test)
  1. Provider A tries to call `save-section` with another provider's ID. Returns 403.
  2. Provider A tries to download another provider's document. Returns 403.

### Admin

- **Create an API key** with scope `providers:read` and verify a GET with that key works.
- **Revoke the key** and verify subsequent requests return 401.

## Accessibility

Every page asserts:

- No axe violations of severity critical or serious.
- Page has exactly one h1.
- Landmarks (`main`, `nav`) are present.
- Images have alt text.
- Focus is trapped inside modals.

## Visual regression (planned)

- Storybook pages captured to Chromatic on PR.
- Baseline per branch.

## Running

```bash
npm run test:e2e
npm run test:e2e:ui
```

## CI

E2E runs in a dedicated job with services (Postgres, Redis, Azurite). Failures upload screenshots, videos, and traces as artifacts. The job fails if any axe rule of critical severity fires.

## Anti-patterns

- Don't test-drive by CSS selector; use `page.getByRole`.
- Don't `waitForTimeout`; use `waitFor`.
- Don't share state across tests; each test creates its own providers.
- Don't skip for a known flake; fix it or quarantine in a dedicated job.
