07
Security & Compliance

HIPAA-Ready by Design

Security isn't an afterthought—it's embedded in every layer of the platform.

  • ✓
    AES-256-GCM Encryption
    SSN, DOB, and all PHI encrypted at the application layer before database storage. Double protection with PostgreSQL at-rest encryption.
  • ✓
    Azure AD Single Sign-On
    All staff authenticate through Essen's existing Azure AD tenant with MFA enforcement. No separate passwords to manage.
  • ✓
    Role-Based Access Control
    5 distinct roles with granular permissions. Specialists, Managers, Committee Members, and Admins each see only what they need.
  • ✓
    Immutable Audit Trail
    Every action logged with timestamp, actor, and before/after values. 10-year retention. No deletions or modifications possible.
  • ✓
    Azure Key Vault for Secrets
    All API keys, portal credentials, TOTP secrets, and encryption keys stored in Azure Key Vault. Zero secrets in code or config files.
  • ✓
    Secure File Storage
    Azure Blob Storage with no public access. Time-limited SAS tokens for staff downloads. RBAC-controlled access via Managed Identity.
  • ✓
    PHI-Safe Bot Operations
    SSN decrypted only in-memory during bot operations (ABFM verification). Never logged, never stored outside the encrypted database field.
  • ✓
    NPDB Confidentiality
    NPDB results restricted to Credentialing Managers and Admins per 45 CFR Part 60 requirements. Separate access tier enforced.
