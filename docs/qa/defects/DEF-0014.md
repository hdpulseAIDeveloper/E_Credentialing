## Root-cause analysis (three compounding causes)

This defect is a **second sighting** of the same product-shape problem
that previously closed as DEF-INFRA-0001 (which added the warmer +
`onDemandEntries` cap). It returned because the prior fix had three
unguarded gaps:

1. **The warmer ignored dynamic routes.** `scripts/dev/warm-routes.mjs`
   filtered `route-inventory.json` with `!r.dynamic`, so the 16 dynamic
   page modules in this app (`/providers/[id]`,
   `/committee/sessions/[id]`, `/medicaid/[id]`,
   `/admin/users/[id]`, `/compliance/[framework]/[id]`, …) compiled
   on the user's first click — and detail pages account for the
   majority of interactive navigation in a CVO product. The 14-second
   spike above is exactly this case.

2. **Webpack was still the dev compiler.** Next.js 14.2 ships
   Turbopack (`next dev --turbo`) as a stable App Router dev compiler
   that is roughly 10× faster than webpack for module compilation.
   The default `dev` script in `package.json` was plain
   `next dev -p 6015`. Even when a route was warmed, any subsequent
   HMR-triggered recompile (every save while iterating) paid the
   webpack cost.

3. **No regression detector.** Pillar S Surfaces 1–6 covered bring-up,
   migrations, sign-in, public surfaces, named-volume staleness, and
   stack-version pin — but not "after warming, are warmed routes
   actually fast?" So the regression was invisible to the gate; only
   the user noticed.

The combination meant: every container recreate (and we did several
during the DEF-0012/0013 fix landing) wiped the Pillar S signal that
nothing was wrong, while the warmer dutifully reported `done:` for
the 52 static routes — leaving 16 dynamic routes uncompiled and
webpack-shaped recompiles for everything else.

## Fix (one root cause per commit, all three landed together)

Per STANDARD.md §4.1 fix-priority ladder, all three fixes are
**production code / configuration** changes, not assertion or
fixture tweaks:

### 1. Turbopack as the default dev compiler

- `package.json`:
  - `"dev": "next dev --turbo -p 6015"` (was `next dev -p 6015`)
  - `"dev:webpack": "next dev -p 6015"` retained as an emergency
    fallback ONLY (and surfaced in §11.3 anti-weakening rule 1).
- `scripts/dev/dev-with-warmup.mjs` (the wrapper that runs in the
  container) now picks `--turbo` by default and logs the choice at
  startup. `FORCE_WEBPACK=1` is the documented one-shot escape hatch;
  using it requires an open defect card (per §11.3).

### 2. Warmer expanded to dynamic routes

- `scripts/dev/warm-routes.mjs` `loadRoutes()` now returns BOTH the
  static and dynamic route lists.
- New `expandDynamicRoutes()`:
  - Resolves each dynamic template's parent list page
    (`/providers/[id]` → `/providers`, `/compliance/[framework]/[id]`
    → `/compliance`).
  - Fetches the list page HTML and harvests the first
    `href="/<parent>/<id>"` match as the sample URL.
  - Falls back to a **synthetic UUID placeholder**
    (`00000000-0000-0000-0000-000000000000`) when no harvestable
    href is found. The dynamic page MODULE compiles regardless of
    whether the loader resolves a row, so a 404 still warms the
    entire route tree.
- The warmer's tail summary now reports both counts:
  `done: 68 routes warmed (52 static + 16 dynamic) in N s`.

### 3. Pillar S Surface 7 — dev-perf invariant

- `scripts/qa/live-stack-smoke.mjs` adds **Surface 7** behind a
  `--dev-perf` flag (off in CI which runs prod builds, on for any
  local `qa:live-stack:full` run).
- Picks a deterministic mix (homepage, dashboard, first 4 staff
  routes from the inventory), authenticates as admin, fetches each
  route TWICE (warm-up discarded, second fetch measured), asserts
  every measured fetch ≤ **2000 ms** (`DEV_PERF_BUDGET_MS`,
  configurable via `--dev-perf-budget`).
- Failure messages name the three most common root causes
  (Turbopack off, warmer skipped, cache invalidated) so the operator
  doesn't have to guess.
- `package.json` exposes the new path:
  - `"qa:live-stack:perf"` — `--dev-perf` only
  - `"qa:live-stack:full"` — both `--volume-probe` AND `--dev-perf`
    (the canonical "fully green local dev" gate)

### 4. Standard documentation

- `docs/qa/STANDARD.md` v1.3.0:
  - new §11 "Dev-loop performance baseline" with the three binding
    requirements, Surface 7 enforcement, anti-weakening clauses,
    and a back-link to this card.
  - new hard-fail condition §4 (15).
- `.cursor/rules/qa-standard.mdc` — workspace mirror with new
  hard-fail #11 and the §11-touching trigger list.
- `~/.cursor/rules/qa-standard-global.mdc` — global mirror, written
  framework-agnostically (Turbopack for Next, default for Vite,
  vite:dev for Remix) so sibling and future HDPulseAI repos inherit
  the standard automatically.
