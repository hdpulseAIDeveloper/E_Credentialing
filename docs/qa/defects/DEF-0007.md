### Coverage gap

The bug went un-noticed during Wave 21 QA because Pillar A (smoke), Pillar B (RBAC), and Pillar E (a11y) all skip the anonymous project (`test.skip(role === null, "pillar-X runs in role projects only")`). No spec under any pillar exercises anonymous reachability of static public routes today. The fix in this card therefore ALSO adds `tests/e2e/anonymous/pillar-a-public-smoke.spec.ts`, which iterates over every `route-inventory.json` entry where `group === "public"` and asserts the route returns 200 to an anonymous client. That spec catches the systemic /legal, /cvo, /sandbox, /pricing, /changelog, /settings/billing, /settings/compliance drift documented separately in **DEF-0008**, while keeping THIS card's production fix scoped to the routes Wave 21 introduced (one root cause per commit, per §4.1 step 4).

## Root-cause analysis

`src/middleware.ts` carries an explicit allow-list of public path prefixes. Wave 21 introduced two new public surfaces (`/errors`, `/errors/[code]`) but did not add either to the allow-list. The middleware therefore falls through to the staff-route branch (`if (!session?.user) → redirect /auth/signin`), and an anonymous visitor never sees the catalog. The page itself is correctly server-rendered (`/errors` is `○ Static`, `/errors/[code]` is `●` SSG with both kebab-case and snake_case URLs pre-rendered for every entry — see the build output in `server.log`); the failure is purely middleware-level.

The deeper root cause — that `src/middleware.ts`'s public allow-list is hand-maintained instead of derived from `route-inventory.json` `group === "public"` — is captured in **DEF-0008** so it can be fixed at the right scope (cross-cutting middleware refactor, separate commit).
