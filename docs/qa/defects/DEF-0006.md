## Captured evidence

axe-core rule `select-name` (impact: critical), 1 node on `/dashboard`,
reported by Pillar E for every staff role. The bare `<select>` (likely
an unwrapped Radix `Select.Trigger` or a native `<select>` filter)
renders without an associated `<label>` / `aria-label` /
`aria-labelledby`.

Full evidence: `…/all-roles-pillar-e-a11y-pillar-E-a11y-dashboard-role-*/error-context.md`.

## Minimum fix

Add an `aria-label` describing the select's purpose (e.g.
`aria-label="Filter providers by status"`), or wrap with a visible
`<label htmlFor=…>`. One-line patch in the dashboard component.

## Fix applied

`src/components/dashboard/PipelineTable.tsx`:

- Added `aria-label="Filter providers by status"` on the `<select>`.
- Added `aria-label="Search providers by name"` on the search `<input>`
  (the placeholder-only input would also have failed `label` /
  `aria-input-field-name` rules under stricter axe profiles).
