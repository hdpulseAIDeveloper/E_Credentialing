# DEF-0012 — `/changelog` returns 500: `.dockerignore` excludes `docs/`, so the runtime cannot read `docs/changelog/public.md` (PRE-EXISTING, surfaced by closing DEF-0008)

## Metadata

- **DEF ID:**            DEF-0012
- **Opened:**            2026-04-19 by Cursor (Composer), surfaced immediately by `npm run qa:live-stack` after the structural fix for DEF-0008/DEF-0011 landed
- **Severity:**          high (a public marketing page returns 500 to anonymous customers; previously hidden by DEF-0008's blanket redirect to `/auth/signin`)
- **Pillar (A–S):**      A — Functional smoke (anonymous), J — API contract (the changelog page is the canonical surface for the `/api/v1/openapi.json` "what changed in this version" link), Q — Documentation integrity (the Wave 5.5 spec promises public release notes), **S — Live-Stack Reality Gate** (the surface that found this)
- **Affected route(s):** `/changelog` (and likely `/changelog.rss`, by the same loader)
- **Affected role(s):**  anonymous (every browser, every prospect)
- **Browser:**           all — server-side 500, browser-agnostic
- **Spec(s) failing:**   `scripts/qa/live-stack-smoke.mjs` Surface 5
- **Status:**            **Open — Awaiting fix** (separate root cause from DEF-0008/DEF-0011 per STANDARD.md §4.1.1; not folded into the structural-fix commit so the structural change stays reviewable)

`/app/docs/` exists in the container but contains ONLY `qa/` — created
implicitly because `docker-compose.dev.yml` bind-mounts
`./docs/qa/inventories:/app/docs/qa/inventories`. Every other `docs/`
subfolder (including `docs/changelog/`, `docs/api/`) is missing.

## Root cause

`.dockerignore` (line 16):

```
docs
```

The image build (`Dockerfile.web`'s `COPY . .` step) silently drops
the entire `docs/` tree. The runtime then fails when:

- `src/lib/changelog/loader.ts` reads `docs/changelog/public.md`
  (`/changelog` page).
- `src/app/api/v1/openapi.json/route.ts` reads
  `docs/api/openapi-v1.yaml` (`/api/v1/openapi.json` and `/api/v1/openapi.yaml`).
- `src/lib/ai/knowledge-base.ts` reads from `process.cwd()` for AI
  chat context grounding.

This bug has shipped to every environment since Wave 5.5 introduced the
runtime markdown loader (2025-Q4). It was hidden in production because:

1. `/changelog` was redirected to `/auth/signin` by the middleware
   (DEF-0008 root cause) — no anonymous customer ever reached the
   page-render code path.
2. `/api/v1/openapi.json` is API-key-gated, so external monitors never
   probed it without credentials, and the internal contract gates run
   against the OpenAPI source on disk in the repo, not against the
   deployed bundle.

Closing DEF-0008 lifted the redirect on `/changelog` and made the
500 visible to the gate on the very first run.

## Why this is filed as a separate card (not folded into the DEF-0008 commit)

Per STANDARD.md §4.1.1 ("one root cause per commit") and per ADR 0028's
anti-weakening rule for Pillar S ("a gate that surfaces three defects
should produce three defect cards, not one — each with its own
reviewable fix"), this is a distinct root cause:

- **DEF-0008 / DEF-0011 root cause:** middleware and inventory used
  separate hand-maintained truth sources for "is this route public."
- **DEF-0012 root cause:** the build context excludes data files the
  runtime needs.

Bundling them would muddy the structural-fix review (the public-routes
single-source-of-truth change is a 100-line structural commit; the
dockerignore fix is a 1-3 line config commit). Keeping them separate
lets each review on its own merits.

## Fix shape (recommended)

### Option 1 (preferred) — narrow `.dockerignore`

Replace the blanket `docs` line with:

```
docs/**
!docs/changelog
!docs/changelog/**
!docs/api
!docs/api/**
```

Includes ONLY the runtime-needed subpaths. `docs/qa/`, `docs/dev/`,
`docs/functional/`, etc. (the heavy documentation tree) stays out
of the image.

### Option 2 — move runtime data files out of `docs/`

Move `docs/changelog/public.md` → `data/changelog/public.md`,
`docs/api/openapi-v1.yaml` → `data/api/openapi-v1.yaml`, update the
loader paths, leave `.dockerignore` untouched. Cleaner conceptually
(docs vs data are different concerns) but a larger blast radius
(documentation references, the `npm run docs:*` scripts, every link
in the per-screen cards). Defer unless the dockerignore fix turns
out to be brittle.

### Option 3 — add bind mounts in `docker-compose.dev.yml`

Add `- ./docs/changelog:/app/docs/changelog` and
`- ./docs/api:/app/docs/api` to the `ecred-web` service. **Fixes dev
only, not prod.** Reject as a standalone fix; could pair with Option 1
if hot-reload of `docs/changelog/public.md` is desired during dev.
