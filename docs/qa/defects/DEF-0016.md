## Root-cause analysis

`src/env.ts` declared the four Azure URL endpoints (and a few other
optional URL fields) as `z.string().url().optional()`. With
`createEnv({ emptyStringAsUndefined: true })` set, an empty value
*is* coerced to `undefined` and the optional contract holds. But
when the operator types a literal placeholder string like
`"placeholder"` (or `"changeme"`, `"TBD"`, `"null"`, `"none"`,
`"<set me>"`, etc.), `.url()` correctly rejects the value, the
schema fails validation, `createEnv` throws, and `next build`
aborts at page-data collection time for every route that
transitively touches `@/env`.

The contract drift is on the schema side, not on the operator
side: a field declared `.optional()` should honor the optional
contract for any non-URL string the operator might write. The
runtime code already handles `undefined` (e.g.
`src/lib/azure/keyvault.ts` returns a no-op client when the URL
is missing). The validation layer was the only thing standing
between a deployable image and a dead build.

This is the same shape of contract drift as DEF-0011
(`route-inventory.json` mis-classifying routes as `group:public`
while runtime middleware disagreed): two layers had different
opinions about what the same field meant.

Why didn't this fire earlier? Same reason as DEF-0015 — the
cold-Dockerfile gate (`npm run qa:dockerfile:cold`) is gated
behind `requiresEnv: true` for `docker-compose.prod.yml` and
reports `NOTRUN` when no production-shaped `.env` is available
locally. The local CI matrix does not exercise the prod compose
file, so the build regression shipped silently.

## Fix-Until-Green attempts

### Attempt 1 — 2026-04-19 19:18 by Cursor

- **Hypothesis:** introduce a typed helper `optionalUrl` in
  `src/env.ts` that preprocesses any string value:
    - trim whitespace,
    - empty → `undefined`,
    - non-`http(s)://` prefix → `undefined`,
    - otherwise pass through to `z.string().url().optional()`.
  Apply it to `NEXTAUTH_URL`, `AZURE_KEY_VAULT_URL`,
  `AZURE_BLOB_ACCOUNT_URL`, `AZURE_DOCUMENT_INTELLIGENCE_ENDPOINT`,
  `AZURE_COMMUNICATION_SERVICES_ENDPOINT`, and
  `STRIPE_BILLING_PORTAL_RETURN_URL`. Do **not** apply it to
  `DATABASE_URL` or `NEXT_PUBLIC_APP_URL`, which are required —
  those keep `z.string().url()` and a placeholder there should
  still fail loudly.

## Forward-looking hardening

This is the second deploy-time defect in the same hour that the
cold-Dockerfile gate would have caught locally if it were not
silently NOTRUN-ing for `docker-compose.prod.yml` (see DEF-0015
§"Forward-looking hardening" item 1). That gap is now urgent,
not just nice-to-have, and is being upgraded to a follow-up PR
target:

> `scripts/qa/check-dockerfile-build.mjs` MUST run the
> `docker-compose.prod.yml` cold build with a synthetic
> `.env.prod.fake` (placeholders for every required key, real
> placeholders for every optional URL — `placeholder`,
> `changeme`, `TBD`) and assert exit 0. NOTRUN must be reserved
> for genuinely environment-impossible cases (e.g. Docker
> daemon not running), not for missing operator config.

Tracked in DEF-0015 §"Forward-looking hardening" item 1 (this
defect adds a second test case the synthetic `.env.prod.fake`
must include: `AZURE_KEY_VAULT_URL=placeholder`).
