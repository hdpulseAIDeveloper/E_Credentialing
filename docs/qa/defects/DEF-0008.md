## Root-cause analysis

`src/middleware.ts` carries an EXPLICIT, hand-maintained list of public path prefixes. The list has fallen behind the ground truth (`docs/qa/inventories/route-inventory.json` where `group === "public"`) at least seven times (DEF-0007 was the eighth). Each new public route requires a paired middleware edit; nothing enforces the pairing.

The right fix shape is a structural one: derive the middleware allow-list from a single source of truth so the route-inventory and the middleware can never disagree. Options the user must pick between:

1. **Generate at build time:** ship a `scripts/qa/build-public-allowlist.ts` that reads `route-inventory.json`, emits `src/middleware.public-routes.generated.ts`, and the middleware imports the generated array. Same shape as `v1-types.ts` / `postman.json`.
2. **Group-folder convention:** move every `group: "public"` page under a `(public)/` route group; the middleware tests `pathname matches one of the (public)/* segments` instead of an explicit list. Refactor blast radius is a `git mv` per route.
3. **Convention-by-route-group + middleware regex:** detect public routes by reading the App Router segment groups at build time (Next.js exposes them in `.next/app-paths-manifest.json`).

All three options are larger than DEF-0007's "one root cause per commit" scope. Recommending the user approve option (1) since it mirrors the SDK / Postman drift gates already in place.

### Attempt 2 — 2026-04-19, Option (1) implemented (CLOSING)

User approved the structural fix in the same conversation that landed Pillar S. Option (1) — single source of truth — was chosen because it most cleanly closes BOTH DEF-0008 (middleware lagging inventory) AND DEF-0011 (inventory ahead of middleware) with a single mechanism, instead of solving them separately and re-introducing drift between the two solutions.

Implementation:

- **`src/lib/public-routes.ts`** (new): exports `PUBLIC_PATH_PREFIXES` (alphabetised array of 14 path prefixes that the middleware allows without a session), `PUBLIC_EXACT_PATHS` (alphabetised array of 7 leaf marketing paths), `isPublicRoute(pathname)` predicate, `isProviderPortalRoute(pathname)` predicate, and the `PROVIDER_PORTAL_PREFIX` constant. The file is dual-consumable from the Edge runtime (middleware) and from Node (the inventory builder script). Anti-weakening rules baked into the file's TSDoc forbid hand-maintained duplicates.
- **`src/middleware.ts`**: replaced the 14-line inline `pathname === "/" || pathname.startsWith("/auth/") || …` allow-list with a single `if (isPublicRoute(pathname)) { … }` branch. Same change for the provider portal: `if (isProviderPortalRoute(pathname)) { … }`. The middleware now has zero hand-maintained route knowledge.
- **`scripts/qa/build-route-inventory.ts`**: replaced the `groupMatch?.[1] ?? "public"` default with a 3-tier `classifyGroup(route, relFile)` helper that uses, in order, (1) the explicit `(group)/` segment if present, (2) `isPublicRoute(route)` for ungrouped pages, (3) `staff` as the default. The OLD code defaulted ALL ungrouped pages to `public`, which is exactly the bug DEF-0011 documents — `/settings/billing` and `/settings/compliance` are not in any `(group)/` folder, so they were declared `public` while the middleware redirected them to signin.
- **`docs/qa/inventories/route-inventory.{md,json}`**: regenerated. The `public` set is now exactly the set the middleware allows (16 entries: `/`, `/auth/register`, `/auth/signin`, `/changelog`, `/cvo`, `/errors`, `/errors/[code]`, `/legal/cookies`, `/legal/hipaa`, `/legal/privacy`, `/legal/terms`, `/pricing`, `/sandbox`, `/verify/carrier/[token]`, `/verify/reference/[token]`, `/verify/work-history/[token]`). `/settings/billing` and `/settings/compliance` correctly classify as `staff`.
