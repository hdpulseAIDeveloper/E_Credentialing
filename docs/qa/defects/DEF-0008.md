## Root-cause analysis

`src/middleware.ts` carries an EXPLICIT, hand-maintained list of public path prefixes. The list has fallen behind the ground truth (`docs/qa/inventories/route-inventory.json` where `group === "public"`) at least seven times (DEF-0007 was the eighth). Each new public route requires a paired middleware edit; nothing enforces the pairing.

The right fix shape is a structural one: derive the middleware allow-list from a single source of truth so the route-inventory and the middleware can never disagree. Options the user must pick between:

1. **Generate at build time:** ship a `scripts/qa/build-public-allowlist.ts` that reads `route-inventory.json`, emits `src/middleware.public-routes.generated.ts`, and the middleware imports the generated array. Same shape as `v1-types.ts` / `postman.json`.
2. **Group-folder convention:** move every `group: "public"` page under a `(public)/` route group; the middleware tests `pathname matches one of the (public)/* segments` instead of an explicit list. Refactor blast radius is a `git mv` per route.
3. **Convention-by-route-group + middleware regex:** detect public routes by reading the App Router segment groups at build time (Next.js exposes them in `.next/app-paths-manifest.json`).

All three options are larger than DEF-0007's "one root cause per commit" scope. Recommending the user approve option (1) since it mirrors the SDK / Postman drift gates already in place.
